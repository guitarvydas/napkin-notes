# Fan-in
a component that receives a message and sends output to more than one destination, or that receives from more than one source, handled naturally rather than as a special case bolted onto a single-output model. This directly refutes the "one-in-one-out" restriction of the functional paradigm.

# Async Compositions Without Tight Coupling
two or more components communicating through messages where neither blocks waiting for the other. The point is not concurrency for its own sake but demonstrating that the components are genuinely topology-blind: each component does not know what it is connected to.
# Probe Insertion

showing that you can add an observational component to the diagram and watch intermediate state, then remove it, without modifying any of the existing components. This is the debugging argument made concrete. It directly demonstrates that the diagrammatic notation preserves design intent in a way that sync/seq debuggers cannot match.

# Container/Leaf distinction
a diagram that contains at least one nested sub-diagram, showing recursive composition. This is what schematics lack and what Lisp-like structure provides. Without this, the demo shows a flat circuit, not a scalable architecture.

# a leaf node doing something sync/seq
— showing that a TPL component (a normal function, a piece of Python or JavaScript) sits naturally at the leaf of an async composition without modification. This validates the synthesis claim: sync is a participant in async, not its replacement.
