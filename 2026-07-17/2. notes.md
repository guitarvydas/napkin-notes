> Markdown for Code

I like this section, but was also thinking that it becomes possible to create sections in code that can be elided in layers, like markdown does with "#", "##", ... headings.

I think that presenting code notations in layers will make onboarding easier and gentler. People can understand a given architecture and its implementation down to any level they care to delve down into.

> Explain it to an LLM

To my understanding Steve Philips did exactly that. He built a new language called [Voltair](https://revolutionize.dev/posts/voltair/) and got it running by having source code be transpiled into Go. At least during the early stages of design.

The language/compiler split pays off regardless of whether you use T2T or an LLM.
