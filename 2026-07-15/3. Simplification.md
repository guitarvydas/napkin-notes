Simplification has its benefits:
1. make it possible to easily transpile to existing languages, like Javascript
2. use _only_ strings, no other data structures (especially user-defined constructs)
3. allow the use of "eval()" during development (same as using compile, but more convenient)

This made it possible to create a little DSL ("SCN" Solution Centric Notation) that could be bolted onto OhmJS to create T2T. OhmJS expects "semantics" code to be written in Javascript. I whipped up a simple SCN, written in OhmJS, called "RWR" to concisely express rewrites as string operations (raw characters + string interpolation), obviating the need to write Javascript. T2T calls OhmJS twice, first to transmogrify the RWR code to JS, then to run the OhmJS along with the generated JS code. JS has an "eval()" function which makes this easy to do (generate JS code, eval it, then run OhmJS again to parse and transmogrify the `st` source code). The T2T code for the `st` example is only about 26+33 lines of code and was written in about an hour and a half.
