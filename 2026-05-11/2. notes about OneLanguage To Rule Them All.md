> The functional paradigm has a precise implicit target: classical physics.

I'm not sure that this captures what I am thinking. SICP's target is a style of thinking that neglects other possibilities. This style has been used to describe classical physics - SICM. Prigogene argues against using this style exclusively. Paul LaViolette (subquantum kinetics) notes that this style is based on thinking in terms of closed systems and he favours using an open system approach. 

I "feel" this difference when I think of programming a single CPU in assembler - it's sequential and synchronous. Yet, when I think in terms of using multiple CPUs, each CPU being just a simple IC - just a "building material" - I end up thinking in terms of asynchronous parallelism where many things can happen at the same time without synchronizing with one another. Each CPU is a closed system which operates in a sequential / synchronous manners, but the whole system composed of many CPUs is not sequential and synchronous and is "open".

The "ultimate" use for SICP is systems like SICM. This requires operating systems and the functional paradigm as scaffolding. Ideas like Forth lead to a more "open systems" approach which does not constrain one to arrive at functional, closed system expressions of physics nor Reprogrammable Machines.

The way I see it, the functional paradigm - SICP, SICM - is but a subset of a broader class of expression. Presently, I think that the broader class cannot be expressed using only one notation, but needs to incorporate many specialized, custom notations. This _should_ lead to expression of irreversible processes and _should_ make it easier to explore ideas like the Brusselator, Subquantum Mechanics, etc.
